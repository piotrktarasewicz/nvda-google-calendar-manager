Google Calendar Manager

Polish version: readme_pl.html

Google Calendar Manager is an NVDA add-on for Windows. It lets you sign in to Google Calendar, choose calendars, read events, and perform basic event operations from the keyboard.

Features

The add-on can:

  • sign in to a Google account through a web browser,
  • check Google Calendar sign-in status,
  • choose which calendars are used by the add-on,
  • read events for today and the next six days,
  • read events for a selected day,
  • search events by text in a selected time range,
  • switch between short and full speech modes,
  • announce events that are currently in progress,
  • skip finished events when reading the current day,
  • add new events, including recurring events,
  • edit existing events,
  • delete one or more selected events.

Scope

Google Calendar Manager is intended for basic calendar access from NVDA. It does not try to replace the full Google Calendar web interface.

The add-on does not manage:

  • meeting participants,
  • Google Meet links,
  • room or participant availability,
  • Google Maps place search,
  • attachments,
  • calendar reminders.

The location field, when used, is plain event text. The add-on does not search Google Maps or verify addresses.

Requirements

  • Windows
  • NVDA 2026.1 or newer
  • Internet access
  • a Google account with Google Calendar enabled

A separate Python installation is not required. The add-on uses the Python environment provided by NVDA and includes the Python libraries it needs.

Installation

  1. Install the add-on in NVDA.
  2. Restart NVDA if required.
  3. Make sure the add-on is enabled.

First sign-in

To sign in to Google Calendar:

  1. Press NVDA+Shift+G, then press 0.
  2. Your default web browser will open.
  3. Sign in to your Google account and allow calendar access.
  4. After sign-in is complete, return to NVDA.
  5. Press NVDA+Shift+G, then press 0 again to check the sign-in status.

If sign-in was successful, the add-on will announce that you are signed in to Google Calendar.

Keyboard shortcuts

First press NVDA+Shift+G to activate the Google Calendar Manager layer. Then press one of the following keys:

Sign-in and status

0 - start Google sign-in or check sign-in status.

Event reading

1 - read events for today.

2 - read events for tomorrow.

3 - read events for the day after tomorrow.

4 - read events for three days from now.

5 - read events for four days from now.

6 - read events for five days from now.

7 - read events for six days from now.

P - choose a day, read events for that day, and open event management actions for that day.

S - search events by text.

Settings

8 - switch between short and full speech mode.

9 - choose which calendars are included.

Event management

N - add an event.

E - edit an event.

U - delete an event.

H - show shortcut help.

Direct shortcuts

The add-on also provides direct shortcuts: NVDA+Control+Shift+N adds an event, NVDA+Control+Shift+E edits an event, and NVDA+Control+Shift+U deletes an event.

Choosing calendars

In the calendar selection window:

  • Up Arrow and Down Arrow move through the list,
  • Space checks or unchecks the selected calendar,
  • Enter saves the selection and closes the window,
  • Escape closes the window without saving changes.

If no calendar is selected, the add-on automatically uses the primary calendar.

Speech modes

Short mode

Short mode reads compact event information.

Examples:

  • 14:30, Meeting
  • until 15:00, Team meeting in progress
  • all-day, Vacation

Short mode is intended for quick calendar checks.

Full mode

Full mode reads more complete event information.

Examples:

  • 14:30 to 15:30, Meeting
  • until 15:00, Team meeting in progress
  • all-day event, Vacation

If multiple calendars are selected, the add-on may also announce the calendar name.

Current-day behavior

For the current day, the add-on:

  • skips events that have already finished,
  • announces events that are currently in progress,
  • reads upcoming events.

Example:

until 15:00, Team meeting in progress

For future days, all events on that day are read.

Searching events

Press NVDA+Shift+G, then S. Enter the text to search for and choose a time range. Available ranges are the next month, next 3 months, next 6 months, next 12 months, or a custom date range.

Search is performed in the selected calendars. Google Calendar may match the entered text in the event title, description, or location. For this reason, the add-on describes this feature as event search, not title-only search.

Search results are shown as a list with event dates and times. From the results window you can start a new search, or choose a found event for editing or deletion. If a found event belongs to a recurring series, the add-on still asks whether the operation should affect only the selected occurrence or the entire series.

Adding events

Press NVDA+Shift+G, then N, or use NVDA+Control+Shift+N. The add-on opens a dialog for the new event.

You can also press NVDA+Shift+G, then P, choose a day, and use the Add event button in the selected-day results window. In that case, the new event dialog is opened with that day already filled in.

You can enter:

  • calendar,
  • title,
  • start and end date,
  • start and end time, unless the event is all-day,
  • location,
  • recurrence.

Supported recurrence options are one time, daily, weekly, monthly, and yearly. A recurring event may have an end date or no recurrence end date.

Editing events

Press NVDA+Shift+G, then E, or use NVDA+Control+Shift+E. Choose the day, select the event, edit the fields, and save changes.

If the selected event belongs to a recurring series, the add-on asks whether to edit only the selected occurrence or the entire series. The option to edit this and following occurrences is not included.

Deleting events

Press NVDA+Shift+G, then U, or use NVDA+Control+Shift+U. Choose the day, select one or more events, then move to the Delete button and confirm the operation.

If one selected event belongs to a recurring series, the add-on asks whether to delete only the selected occurrence or the entire series. If several selected events include recurring events, the add-on asks once and applies the chosen scope to all recurring events in that selection. The option to delete this and following occurrences is not included.

Add-on language

The add-on uses the current NVDA language for its own messages and dialogs.

Currently supported:

  • English
  • Polish

If the current NVDA language is not supported, the add-on uses English. Event titles, descriptions, and locations are user content from Google Calendar and are not translated by the add-on.

User files

The add-on may create the following local files:

  • token.json - local Google sign-in token,
  • settings.json - selected calendars and speech mode settings,
  • last_oauth_error.txt - helper file for OAuth error diagnostics.

These files are stored locally in the NVDA user configuration directory, in the googleCalendarManager subfolder. When updating from older test versions, the add-on can copy existing files from the previous googleCalendarReader subfolder or from the add-on directory if the current files are not already present.

Privacy

The add-on uses the Google Calendar API to read and modify calendar events according to the user's actions. It requests only the Calendar API scopes needed for its current features:

  • https://www.googleapis.com/auth/calendar.events - read and manage events in the user's calendars,
  • https://www.googleapis.com/auth/calendar.calendarlist.readonly - read the user's calendar list so calendars can be selected in the add-on,
  • https://www.googleapis.com/auth/calendar.settings.readonly - read Calendar settings such as the calendar time zone.

The add-on does not request permission to create, delete, share, or change calendars themselves.

The add-on does not:

  • send calendar data to servers controlled by the author,
  • share calendar data with third-party services controlled by the author,
  • use Google Calendar data for advertising or analytics.

Calendar data is processed locally by the add-on and exchanged with Google only as needed for Google Calendar API operations. The Google sign-in token is stored locally in the NVDA user configuration directory.

Troubleshooting

The add-on says the user is not signed in

Press NVDA+Shift+G, then 0, and complete sign-in in the web browser.

The browser opens but sign-in does not complete correctly

Check the Internet connection and make sure the Google account has access to Google Calendar.

The add-on reads from the wrong calendar

Press NVDA+Shift+G, then 9, and check which calendars are selected.

The add-on reads too much or too little information

Press NVDA+Shift+G, then 8, to switch the speech mode.

An add, edit, or delete operation fails

Check the error details shown by the add-on. Also make sure the selected calendar allows the operation. Some shared calendars may allow reading but not editing.
